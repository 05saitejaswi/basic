Why we need a library/framework at all
If you build a real web app with just plain JS/HTML/CSS, you'll quickly run into the same painful problems over and over:
Problem with plain JSWhat a framework gives youManually updating the DOM every time data changes (slow, buggy, easy to forget a spot)Reactive rendering — UI updates automatically when data changesNo standard way to structure code as app growsComponent-based architecture — reusable, isolated pieces of UIRe-writing the same patterns (forms, routing, state) on every projectBuilt-in tooling: routing, state management, forms, HTTP handlingHard to keep large teams consistentConventions & ecosystem — shared patterns, libraries, community solutionsPerformance issues from re-rendering whole pagesVirtual DOM / fine-grained reactivity — only update what actually changedTesting is messy without structureTestability — components are easier to unit test in isolation
So the short version: frameworks solve the scaling problem. Plain JS is fine for a simple page; once you have dynamic data, many UI states, and a growing team, you need structure — that's what React/Angular/Vue give you.
